| Код   | Требование                                                                                                     | Комментарий                                                                                                       |
|-------|----------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------|
| F     | Функциональные (Functionality)                                                                                 |                                                                                                                   |
| F-01  | Отображение актуальных ставок депозитов на сайте и в интернет-банке                                            | Требуется централизованное хранилище ставок вместо XLS-файлов; необходимо обеспечить синхронизацию между каналами |
| F-02  | Персонализация ставок депозитов в интернет-банке для авторизованных клиентов                                   | Требует интеграции с системой профилирования клиента и механизмом расчёта персональных условий                    |
| F-03  | Подача заявки на депозит через сайт (ФИО, телефон) с последующим звонком менеджера                             | Данные передаются в систему кол-центра; требуется шифрование персональных данных при передаче                     |
| F-04  | Подача заявки на депозит через интернет-банк с СМС-подтверждением                                              | Интеграция с ядром СМС-функционала; операция должна быть идемпотентной                                            |
| F-05  | Подача заявки на кредит через сайт с возможностью предварительного скоринга по БКИ                             | Требует интеграции с системой скоринга; для новых клиентов используется только общедоступная информация БКИ       |
| F-06  | Отображение предодобренных кредитных предложений в интернет-банке                                              | Необходимо хранение результатов предыдущих скорингов и механизм их актуализации                                   |
| F-07  | Единый жизненный цикл заявки (сайт → отделение → АБС)                                                          | Требуется уникальный идентификатор заявки и механизм её восстановления в АБС по данным клиента                    |
| F-08  | Отправка СМС-уведомлений о статусе заявки (подтверждение ставки, открытие депозита, изменение статуса кредита) | Интеграция с ядром СМС; необходимо избежать доработки подрядчиком — реализовать через внутренние сервисы          |
| F-09  | Обязательная идентификация новых клиентов в отделении перед открытием продукта                                 | Регуляторное требование; архитектурно влияет на проектирование воронки продаж                                     |
| U     | Удобство использования (Usability)                                                                             |                                                                                                                   |
| U-01  | Единая система дизайна во всех каналах (сайт, интернет-банк) с корпоративным брендированием                    | Требует централизованной библиотеки компонентов и механизма их синхронизации между фронтендами                    |
| U-02  | Время отклика интерфейса ≤ 100 мс для всех пользовательских операций                                           | Критично для конверсии; требует оптимизации загрузки справочных данных (см. проблему >1 с)                        |
| U-03  | Минимизация шагов при подаче заявки (особенно для предодобренных кредитов)                                     | Влияет на архитектуру форм: необходимость предзаполнения данных из профиля клиента                                |
| R     | Надёжность (Reliability)                                                                                       |                                                                                                                   |
| R-01  | Доступность сервисов 99,9% (24/7) с автоматическим переключением на резервный ЦОД                              | Требует реализации паттерна Circuit Breaker и механизма кворума для распределённых операций                       |
| R-02  | Отказоустойчивость интернет-банка при недоступности АБС                                                        | Необходима асинхронная обработка заявок через очереди; АБС не должен быть точкой отказа                           |
| R-03  | Гарантированная доставка СМС-уведомлений даже при временных сбоях                                              | Требует механизма повторных попыток и хранения статуса доставки в отказоустойчивом хранилище                      |
| R-04  | Сохранность данных заявок при сбоях в процессе обработки                                                       | Необходима реализация Saga pattern или двойной записи с компенсирующими транзакциями                              |
| P     | Производительность (Performance)                                                                               |                                                                                                                   |
| P-01  | Время обработки операции подтверждения СМС ≤ 500 мс                                                            | Критично для пользовательского опыта; требует кэширования сессий подтверждения                                    |
| P-02  | Загрузка справочных данных (ставки, условия) ≤ 100 мс                                                          | Требует многоуровневого кэширования (Redis + CDN) и фоновой предзагрузки                                          |
| P-03  | Горизонтальное масштабирование сервисов интернет-банка и кредитного конвейера                                  | Возможность добавления инстансов без простоя; требует stateless-архитектуры                                       |
| P-04  | Минимизация нагрузки на систему скоринга (запрет предрасчётов в пиковую нагрузку)                              | Требует очередей с приоритетами и ограничения частоты вызовов (rate limiting)                                     |
| P-05  | Равномерное распределение нагрузки между ЦОД и серверами                                                       | Требует интеллектуального балансировщика с учётом географии клиентов и состояния ЦОД                              |
| S     | Поддерживаемость (Supportability)                                                                              |                                                                                                                   |
| S-01  | Полная документация архитектуры, API и процессов для дальнейшего расширения                                    | Критично для перехода на микросервисную архитектуру; требуется ведение в актуальном состоянии                     |
| S-02  | Использование существующих СУБД банка (MS SQL, Oracle) для новых сервисов                                      | Снижение операционных рисков; упрощение администрирования и резервного копирования                                |
| S-03  | Предпочтение Kafka для асинхронной коммуникации (с учётом будущей миграции интернет-банка)                     | Несмотря на несовместимость текущей версии, необходимо проектировать с учётом будущего перехода                   |
| S-04  | Микросервисная архитектура для новых компонентов (депозиты/кредиты) с чёткими границами                        | Позволит изолировать нагрузку от АБС; упростит горизонтальное масштабирование                                     |
| S-05  | Централизованное логирование и мониторинг сквозных транзакций (distributed tracing)                            | Необходимо для диагностики проблем в распределённой системе                                                       |
| S-06  | Автоматизированное тестирование критических сценариев (особенно СМС-подтверждение)                             | Снижение рисков при развёртывании в условиях 24/7 доступности                                                     |
| +R    | + Ограничения (Restrictions)                                                                                   |                                                                                                                   |
| +R-01 | Запрет прямых синхронных вызовов API АБС из интернет-банка                                                     | БД АБС перегружена; любая интеграция должна быть асинхронной через промежуточный слой                             |
| +R-02 | АБС поддерживает только вертикальное масштабирование                                                           | Ограничивает возможности роста нагрузки; требует строгого контроля количества обращений                           |
| +R-03 | Обмен данными между АБС и Кредитным конвейером возможен только раз в сутки                                     | Архитектурное ограничение; требует проектирования с учётом асинхронной обработки заявок                           |
| +R-04 | Версия платформы интернет-банка несовместима с современными очередями (Kafka)                                  | Требует промежуточного решения (например, REST + БД как очередь) до миграции                                      |
| +R-05 | Обязательное шифрование всех каналов передачи персональных данных (TLS 1.2+)                                   | Регуляторное требование ЦБ РФ; влияет на выбор протоколов интеграции                                              |
| +R-06 | Запрет хранения персональных данных новых клиентов до идентификации в отделении                                | Требование 152-ФЗ; архитектурно влияет на жизненный цикл заявки                                                   |
| +R-07 | Наличие внутренней экспертизы по используемым технологиям (языки, фреймворки)                                  | Ограничение на выбор стека; исключает «модные», но неподдерживаемые технологии                                    |
| +R-08 | Совместимость новых решений с существующими платформами разработки банка                                       | Требует анализа текущего ландшафта перед выбором технологий                                                       |
